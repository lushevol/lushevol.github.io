IOS12 beta升级方法

发表于 2018-08-20

  1. 前言

    截至这篇文章(2018.08.20)写的时候，IOS12 beta已经发布了第八个测试版本，也即是IOS12 beta8。上个版本beta7由于出现重大bug而被紧急撤回了，导致版本迅速变成了8。

  2. 升级方法

    废话少说，我们来升级吧。我的手机是iphone6s，是IOS12适配的机型之一（最低适配5S），升级之前的系统是11.4。IOS12 beta并不能通过常规方式进行直接升级，而是需要先下载描述文件到iphone，才能触发系统的升级选项。

    • 首先用iphone的浏览器打开描述文件网址

      https://beta.apple.com/sp/betaprogram/redemption?locale=zh#ios

      浏览器会提示登录，用apple id登录。

      网页里已经标好了一二三步，第一步用iTunes备份系统，已经有iCloud云备份的就不用额外备份了，如果没有的化，需要在电脑上备份，具体方法就不多说了，备份系统包括应用数据，会比较大（我的备份就有17GB…）。

    • 第二步下载描述文件，系统会提示“此网站正尝试打开‘设置’以向您显示一个配置描述文件。您允许吗？”，点击允许，下载描述文件。

    • 安装描述文件，输入密码，同意一长串协议后安装。安装完成提示重启。

    • 重启后在“通用” -> “软件更新” 中就能发现IOS12 beta的更新提示了！

    到此，IOS12 beta已经可以在你的手机上更新了，不要忘了备份数据哦~

  3. 后话

    我是听说IOS12有着对老设备完美的兼容性和巨大的性能提升才下决心升级的，据媒体称“升级之后，手机如丝般顺滑”。额，但是怎么说呢，经过快一个月的试用，我并没发现比IOS11.4快很多，APP确实在开启时间和等待时间上缩减了不少，但响应速度还是慢得出奇。而且不知道是不是试用版的原因（或者是手机真的跟不上了），经常会出现APP内容刷不出来的情况。有时候也会出现图标重叠的bug，然而无关紧要。

    现在对IOS12最大的希望就是赶紧发布正式版，毕竟beta版还有很多bug，性能优化估计就到这儿了。

flutter 入门教程之从零开始搭建flutter

发表于 2018-08-18 | 更新于 2018-08-20

  1. 准备工作

    • 系统：笔者使用的是Windows 10系统，官网要求至少是Windows 7及以上。
    • git：flutter SDK开源，托管在github上，因此持续开发需要从github上拉取代码。
    • Java SDK: 下载安装。选默认的就行。
    • android studio & android SDK: 需要安装安卓IDE和SDK，并下载Dart和flutter组件。之前没有用过的同学会遇到一些麻烦，解决办法见文末。
    • 其他：不能科学上网也没关系，贴心的谷歌为国内用户开放了国内镜像。

  2. 起步

    让我们开始吧！

    首先下载flutter代码，从github上拉取或者直接下载压缩包都行。这里演示github下载

    注意：这里使用beta分支，因为目前master版本依赖的SDK版本很新，国内镜像没有更新。

    然后配置环境变量，在系统变量中

    • 新增变量 PUB_HOSTED_URL 值为 https://pub.flutter-io.cn //国内镜像
    • 新增变量 FLUTTER_STORAGE_BASE_URL 值为 https://storage.flutter-io.cn //国内镜像
    • 将flutter/bin添加到PATH中

    重启计算机，双击flutter目录中的 flutter_console.bat 文件，开启一个命令终端。

    在终端中输入flutter doctor，检测环境是否安装完全（如果已经按照前面几步完成的化，这里只会出现一个 No devices available 错误）。

  3. 启动

    下面让我们来开始体验flutter的应用开发吧 : )

    • 在Android Studio启动页选择 Start a new Flutter project
    • 点击flutter application
    • 选择路径下载flutter SDK，然后再次选择该目录下的flutter(真·SDK路径，否则会报 Flutter SDK is not found in the specified location)
    • 选择工程路径，修改工程名称和描述。点击下一步。
    • 根据自己的情况选择是否勾选Kotlin或IOS。点击finish。

    一切完成后，会出现包含Material 组件的示例代码。但我们还缺少Android虚拟机，因此我们来创建一个。

    • 点击顶部导航栏Tools -> AVD Manager
    • 选择Create Virtual Device，在提供的虚拟设备中选择一个（我选择的是Pixel 2 XL）。
    • 选择Android系统版本，并点击Download下载镜像。
    • 完成后运行虚拟设备。一个Android虚拟机就被创建出来了！

    此时点击Android Studio的Run按钮，等待编译，便可在虚拟机中看到示例代码的运行结果了。

  4. 可能出现的问题

    • android studio 和 SDK下载

      Android studio 下载地址，下载完成后配置proxy：Configure -> settings -> Appearance & Behavior -> System Settings -> HTTP Proxy，选中Auto-detect proxy settings，勾选下方Automatic proxy configuration URL，填入代理地址 mirrors.neusoft.edu.cn:80（大连东软信息学院镜像服务器，任意镜像服务器都可以） 。然后点击HTTP Proxy下方的Android SDK，首先配置路径，然后会自动要求下载各种SDK，直接下一步即可。

      如果没有开发过项目，flutter会提示你没有同意Android的license，这个时候直接执行flutter doctor --android-licenses后一路同意即可。

    • Dart和flutter组件

      在Configure -> Plugins中搜索Dart和flutter（标准库中没有这俩，需要Browse Repositories中搜索），安装后点击Restart Android Studio 。

    • Emulator: emulator: ERROR: x86_64 emulation currently requires hardware acceleration!

      选择Tools -> SDK Manager，选择SDK Tools -> intel x86 Emulator Accelerator (HAXM installer)，点击Apply。它会让你下载，等待下载并finish即可。
