语法转义工具 - babel

Babel其实是一个编译JavaScript的平台，它可以编译代码帮你达到以下目的：

  • 让你能使用最新的JavaScript代码（ES6，ES7…），而不用管新标准是否被当前使用的浏览器完全支持；
  • 让你能使用基于JavaScript进行了拓展的语言，比如React的JSX；

配置

Babel本质上是几个模块化的包，其核心功能位于称为babel-core的npm包中，webpack可以把其不同的包整合在一起使用，对于每一个你需要的功能或拓展，你都需要安装单独的包（用得最多的是解析Es6的babel-env-preset包）。

SCSS 语法规范

Sass 有两种语法规则(syntaxes),目前新的语法规则（从 Sass 3开始）被称为 “SCSS”( 时髦的css（Sassy CSS）),它是css3语法的的拓展级，就是说每一个语法正确的CSS3文件也是合法的SCSS文件，SCSS文件使用.scss作为拓展名。第二种语法别成为缩进语法（或者 Sass），它受到了Haml的简洁精炼的启发，它是为了人们可以用和css相近的但是更精简的方式来书写css而诞生的。它没有括号，分号，它使用 行缩进 的方式来指定css 块，虽然sass不是最原始的语法，但是缩进语法将继续被支持，在缩进语法的文件以 .sass 为拓展名。

注释

有三种形式：

（1）//comment：该注释只是在.scss源文件中有，编译后的css文件中没有。

（2）/! /：重要注释，任何style的css文件中都会有，一般放置css文件版权说明等信息。

（3）/ /：该注释在compressed的style的css中没有，其他style的css文件都会含有。

　　运行命令：sass –style compressed test.scss test.css，即可生成压缩版的css文件，并且命名为test.css。几点说明：

3.侦听文件和文件夹

如果希望某一个scss文件或者相应的文件夹下面文件修改后，自动进行编译，那么可以使用侦听命令。

（1）侦听文件：

sass –watch –style compressed test.scss:test.css

当test.scss文件有修改后，会自动编译为test.css，并且是compressed的。

（2）侦听文件夹：

sass –watch –style compressed sass:css

当sass文件夹下.scss文件有修改的时候，会自动编译为与sass中文件同名的css文件。

（1）注意源文件和目标文件之间是冒号，与编译命令中为空格不同。

TCP连接复用

发表于 2019-03-13 | 更新于 2018-08-21

这是我16年调研负载均衡时记录的内容，把它翻出来看看。

TCP MULTIPLEXING

负载均衡器与后台真实服务器的连接复用，即有空闲的长连接就使用空闲的长连接，没有就建立新的连接然后通讯结束之后不关闭，下次请求直接使用该空闲连接。避免了新建TCP连接造成的延时和资源消耗。

采用TCP连接复用技术后，客户端（如：ClientA）与负载均衡设备之间进行三次握手并发送HTTP请求。负载均衡设备收到请求后，会检测服务器是否存在空闲的长连接，如果不存在，服务器将建立一个新连接。当HTTP请求响应完成后，客户端则与负载均衡设备协商关闭连接，而负载均衡则保持与服务器之间的这个连接。当有其它客户端（如：ClientB）需要发送HTTP请求时，负载均衡设备会直接向与服务器之间保持的这个空闲连接发送 HTTP请求，避免了由于新建TCP连接造成的延时和服务器资源耗费。

  • 由此可见，TCP连接复用是由负载均衡器完成的工作，前提是TCP连接不关闭，负载均衡器寻找空闲的连接或者（满员情况下）建立新连接。

连接复用效率的关键在于负载均衡设备是否能够及时释放已经空闲的服务器端连接。最好的连接复用方法在于负载均衡设备给客户端发送HTTP响应之后，收到客户端确认ACK数据报即释放该连接，这种方法避免了任何方式的等待时间，理论上没有更高效的复用方法。（释放TCP连接还是HTTP连接？）

一般拥有tcp连接复用的系统称之为NetScaler，即网络加速器，或者Application Delivery Control，即应用交付控制器。这种设备拥有一个Connection reuse pool（连接复用池），当客户端请求到达时，检查pool中是否存在可用连接，有的话便用作传输连接；没有的话则创建新的连接传输。传输结束后，返回pool中，等待该客户端或者其他客户端的请求。

值得注意的是，相同客户端的请求不一定使用相同的连接。

TCP连接复用不仅提供了http协议的复用，也提供了SSL的复用（避免CPU对每次连接都进行密钥交换）和数据流的复用（即是会话保持，同一个客户端的请求发送给同一个后台）。

TCP_TW_REUSE

复用处于TIME_WAIT状态的端口，使其立刻投入到连接的使用中。

注意：只有认定为”安全”的端口才会被reuse，有两个要求：

  1. tcp_timestamps和tcp_tw_reuse都必须打开；
  2. 重用TIME_WAIT条件是端口最后接收的报文超过了1s时间

个人认为这和TCP连接复用不一样:

TCP连接复用是不关闭已经处理完事务的TCP连接，再次使用它们通讯，即TCP连接未断开，属于LoadBalancer的特性。

而TCP_TW_REUSE则是真实服务器的socket使用与否的问题，即真实服务器启用该内核参数后，会快速重新使用TIME_WAITE端口，使得连接队列不容易拥塞，属于服务器的特性。

HTTP REUSE

HTTP连接复用： 在 HTTP 1.0中，客户端的每一个HTTP请求都必须通过独立的TCP连接进行处理，而在HTTP 1.1中，对这种方式进行了改进。客户端可以在 一个TCP连接 中发送多个HTTP请求，这种技术叫做HTTP复用（HTTP Multiplexing）。它与TCP连接复用最根本的区别在于，TCP连接复用是将 多个客户端 的HTTP请求复用到一个服务器端TCP连接上，而 HTTP复用则是 一个客户端的多个HTTP请求 通过一个TCP连接进行处理。前者是负载均衡设备的独特功能；而后者是HTTP 1.1协议所支持的新功能，目前被大多数浏览器所支持。

IOS12 beta升级方法

发表于 2019-03-13 | 更新于 2018-08-20

  1. 前言

    截至这篇文章(2018.08.20)写的时候，IOS12 beta已经发布了第八个测试版本，也即是IOS12 beta8。上个版本beta7由于出现重大bug而被紧急撤回了，导致版本迅速变成了8。

  2. 升级方法

    废话少说，我们来升级吧。我的手机是iphone6s，是IOS12适配的机型之一（最低适配5S），升级之前的系统是11.4。IOS12 beta并不能通过常规方式进行直接升级，而是需要先下载描述文件到iphone，才能触发系统的升级选项。

    • 首先用iphone的浏览器打开描述文件网址

      https://beta.apple.com/sp/betaprogram/redemption?locale=zh#ios

      浏览器会提示登录，用apple id登录。

      网页里已经标好了一二三步，第一步用iTunes备份系统，已经有iCloud云备份的就不用额外备份了，如果没有的化，需要在电脑上备份，具体方法就不多说了，备份系统包括应用数据，会比较大（我的备份就有17GB…）。

    • 第二步下载描述文件，系统会提示“此网站正尝试打开‘设置’以向您显示一个配置描述文件。您允许吗？”，点击允许，下载描述文件。

    • 安装描述文件，输入密码，同意一长串协议后安装。安装完成提示重启。

    • 重启后在“通用” -> “软件更新” 中就能发现IOS12 beta的更新提示了！

    到此，IOS12 beta已经可以在你的手机上更新了，不要忘了备份数据哦~

  3. 后话

    我是听说IOS12有着对老设备完美的兼容性和巨大的性能提升才下决心升级的，据媒体称“升级之后，手机如丝般顺滑”。额，但是怎么说呢，经过快一个月的试用，我并没发现比IOS11.4快很多，APP确实在开启时间和等待时间上缩减了不少，但响应速度还是慢得出奇。而且不知道是不是试用版的原因（或者是手机真的跟不上了），经常会出现APP内容刷不出来的情况。有时候也会出现图标重叠的bug，然而无关紧要。

    现在对IOS12最大的希望就是赶紧发布正式版，毕竟beta版还有很多bug，性能优化估计就到这儿了。

flutter 入门教程之从零开始搭建flutter

发表于 2019-03-13 | 更新于 2018-08-20

  1. 准备工作

    • 系统：笔者使用的是Windows 10系统，官网要求至少是Windows 7及以上。
    • git：flutter SDK开源，托管在github上，因此持续开发需要从github上拉取代码。
    • Java SDK: 下载安装。选默认的就行。
    • android studio & android SDK: 需要安装安卓IDE和SDK，并下载Dart和flutter组件。之前没有用过的同学会遇到一些麻烦，解决办法见文末。
    • 其他：不能科学上网也没关系，贴心的谷歌为国内用户开放了国内镜像。

  2. 起步

    让我们开始吧！

    首先下载flutter代码，从github上拉取或者直接下载压缩包都行。这里演示github下载

    注意：这里使用beta分支，因为目前master版本依赖的SDK版本很新，国内镜像没有更新。

    然后配置环境变量，在系统变量中

    • 新增变量 PUB_HOSTED_URL 值为 https://pub.flutter-io.cn //国内镜像
    • 新增变量 FLUTTER_STORAGE_BASE_URL 值为 https://storage.flutter-io.cn //国内镜像
    • 将flutter/bin添加到PATH中

    重启计算机，双击flutter目录中的 flutter_console.bat 文件，开启一个命令终端。

    在终端中输入flutter doctor，检测环境是否安装完全（如果已经按照前面几步完成的化，这里只会出现一个 No devices available 错误）。

  3. 启动

    下面让我们来开始体验flutter的应用开发吧 : )

    • 在Android Studio启动页选择 Start a new Flutter project
    • 点击flutter application
    • 选择路径下载flutter SDK，然后再次选择该目录下的flutter(真·SDK路径，否则会报 Flutter SDK is not found in the specified location)
    • 选择工程路径，修改工程名称和描述。点击下一步。
    • 根据自己的情况选择是否勾选Kotlin或IOS。点击finish。

    一切完成后，会出现包含Material 组件的示例代码。但我们还缺少Android虚拟机，因此我们来创建一个。

    • 点击顶部导航栏Tools -> AVD Manager
    • 选择Create Virtual Device，在提供的虚拟设备中选择一个（我选择的是Pixel 2 XL）。
    • 选择Android系统版本，并点击Download下载镜像。
    • 完成后运行虚拟设备。一个Android虚拟机就被创建出来了！

    此时点击Android Studio的Run按钮，等待编译，便可在虚拟机中看到示例代码的运行结果了。

  4. 可能出现的问题

    • android studio 和 SDK下载

      Android studio 下载地址，下载完成后配置proxy：Configure -> settings -> Appearance & Behavior -> System Settings -> HTTP Proxy，选中Auto-detect proxy settings，勾选下方Automatic proxy configuration URL，填入代理地址 mirrors.neusoft.edu.cn:80（大连东软信息学院镜像服务器，任意镜像服务器都可以） 。然后点击HTTP Proxy下方的Android SDK，首先配置路径，然后会自动要求下载各种SDK，直接下一步即可。

      如果没有开发过项目，flutter会提示你没有同意Android的license，这个时候直接执行flutter doctor --android-licenses后一路同意即可。

    • Dart和flutter组件

      在Configure -> Plugins中搜索Dart和flutter（标准库中没有这俩，需要Browse Repositories中搜索），安装后点击Restart Android Studio 。

    • Emulator: emulator: ERROR: x86_64 emulation currently requires hardware acceleration!

      选择Tools -> SDK Manager，选择SDK Tools -> intel x86 Emulator Accelerator (HAXM installer)，点击Apply。它会让你下载，等待下载并finish即可。
